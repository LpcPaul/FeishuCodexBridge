# FeishuCodexBridge 项目说明

FeishuCodexBridge 是一个飞书到 Codex CLI 的本机桥接服务。

它的目标用户是：已经在本机使用 Codex，并希望通过飞书私聊或群聊把任务交给本机 Codex 的用户。

## 当前版本

`0.4.1`

## 产品边界

本项目主要做四件事：

1. 连接飞书机器人消息和本机 Codex CLI。
2. 管理飞书主 Bot 长期聊天里的上下文和话题边界。
3. 让 Codex 可以按需声明飞书交互卡片，默认用 CardKit 2.0 发送表单卡片，并把按钮点击或表单提交续回同一会话。
4. 可选把复杂长内容创建为飞书文档，并把文档链接发回聊天。

本项目不做：

- Codex 执行过程可视化。
- 全量结果反馈卡模板系统。
- 飞书文档协作平台。
- 飞书表格、多维表格、任务系统的内置能力。
- 独立 Agent 路由平台。

## 用户需要提供什么

安装时用户只需要提供飞书自建应用机器人的：

- `App ID`
- `App Secret`

飞书机器人创建与权限配置见：

```text
docs/feishu-app-setup.md
```

## 默认上下文策略

- 私聊主 Bot 默认继续当前话题。
- 超过 2 小时无互动，且没有飞书触发的 Codex 任务正在运行时，下一条消息自动进入新话题。
- 新话题开启时用飞书卡片提示用户，并提供继续上个话题的按钮。
- Codex 任务执行期间不会因为超过 2 小时而切新话题。
- 任务完成后，才重新计算 2 小时空闲时间。

## 默认回复策略

- 收到任务消息后，先回复：`收到，我要开始干活了，稍等我`。
- Codex 最终返回普通文本后，Bridge 直接把文本发回飞书。
- Codex 最终返回 `feishu-card` 或 `feishu_doc` 声明时，Bridge 负责转换成飞书卡片或飞书文档。
- 每次转发给 Codex 的 prompt 都附加移动端回复格式要求，避免大段长文。

## 运行形态

当前主安装形态是：

```text
GitHub 开源项目 + macOS launchd 本机常驻服务
```

MCP 和 Codex Plugin 不是当前主形态。MCP 更适合未来给 Codex 增加飞书资源工具；本项目的核心是监听飞书事件并主动唤起 Codex CLI。
